convert_parameters(dynamicParameters: (double[]) Free parameters, considered: (bool[]) Which parameters are free)[source]¶

Converts an array of given parameters to an array of standard (maximal) length and in the parameter domain of the model

Not all parameters may be considered in the model (to avoid overfitting) Furthermore, some parameters must be constrained to be positive or within a certain interval. In this method, the parameter vector (containing only the values of the free parameters) is transformed to a vector in the parameter space of the model

dynamicParameters contains the parameters that are not held constant considered is true at the entries corresponding to the parameters that

are free. considered must have exactly as many True entries as the length of dynamicParameters

get_mean_factor(params: (double[]) Parameters, considered: (bool[]) Which parameters are free, pair: ((int, int)) Source-sink pair for which the factor shall be determined = None)[source]¶

Returns a factor proportional to the mean traveller flow between the source-sink pair pair or all sources and sinks (if pair is None)

params contains the free model parameters considered is true at the entries corresponding to the parameters that

are free. considered must have exactly as many True entries as the length of dynamicParameters

pair is the source-sink pair of interest (the indices of the source and

the sink, NOT their IDs. If None, the factors for all source-sink combinations are computed)

This method MUST be overwritten. Otherwise the model will through an error.

get_mean_factor_autograd(params, considered)[source]¶

Same as get_mean_factor, but must use autograd’s functions instead of numpy. (Simply exchange np.[…] with ag.[…]) This function is necessary to compute derivatives with automatic differentiation.

Note that autograd functions so not support in-place operations. Therefore, an autograd-compatible implementation may be less efficient. If efficiency is not of greater concern, just use the autograd functions in get_mean_factor already and leave this method untouched.

set_sources_considered(considered: (bool[]) Array determining which of the sources are considered = None, infested: (bool, optional) Select considered sources based on the infestation status = None)[source]¶

Sets which origins are considered in the model fit.

It can happen that the model is to be fitted to model boaters coming from specific jurisdictions, e.g. a model for long-distance traffic and a model for intermediate-distance traffic. In this case, considered can be used to specify the origins considered to fit the model. If infested is given and considered is NOT specified, then the model will be fitted to the jurisdictions with the given infestation status. E.g. infested=True will result in the model being fitted to estimate traffic from infested jursidictions only. All other data will be ignored.

find_potential_routes(stretchConstant: (>=1) Maximal length of the admissible paths in relation to shortest path = 1.5, localOptimalityConstant: ([0, 1]) Fraction of the path that must be optimal = 0.2, acceptionFactor: ((0, 1]) Relaxation factor for local optimality constraint = 0.6666666666666666, rejectionFactor: ([1, 2]) False rejection factor for local optimality constraint = 1.3333333333333333)[source]¶

Searches potential routes of boaters.

For detailed documentation on the arguments, refer to FlowPointGraph.find_alternative_paths

stretchConstant: controls the length of the admissible paths. 1 refers

to shortest paths only, infinity to all paths.

localOptimalityConstant: controls how optimal the admissible paths shall

be. 0 refers to all paths, 1 refers to shortest paths only.

acceptionFactor: approximation factor to speed up the local optimality

check. 0 accepts all paths, 1 performs an exact check. Choose the largest feasible value. 1 is often possible.

rejectionFactor: approximation factor to speed up the local optimality

check. 1 performs exact checks, 2 may reject paths that are admissible but not locally optimal twice as much as required. Choose the smallest feasible value. 1 is often possible.

Preprocesses the graph to make path search queries more efficient
